# 07 Maintainability
## Purpose
Collect evidence that code quality and delivery workflows are sustainable.
Use this folder for CI, coverage, and test-asset tracking.
## Expected artifacts
- CI run evidence in ci/
- Coverage outputs in coverage/
- Test catalogs in 	est-inventory/
## Naming convention
Use YYYY-MM-DD_<artifact>.<ext>, for example 2026-03-10_ci-summary.md.
## Redaction reminder
Remove secrets from logs and redact internal-only links where required.
