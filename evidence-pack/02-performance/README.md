# 02 Performance Efficiency
## Purpose
Collect objective performance evidence for responsiveness and load behavior.
Use this folder for benchmark exports and run logs.
## Expected artifacts
- Lighthouse exports in lighthouse/
- k6 result files in k6/
- Runtime or test logs in logs/
## Naming convention
Use YYYY-MM-DD_<artifact>.<ext>, for example 2026-03-10_lighthouse-home.html.
## Redaction reminder
Remove secrets, internal URLs, and sensitive identifiers before committing artifacts.
