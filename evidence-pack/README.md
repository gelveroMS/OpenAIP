# ISO Evidence Pack

ISO/IEC 25010:2023 Evidence Pack for IT Evaluation.

## Rules

- All artifacts must reference the same evaluation build metadata: commit hash or tag, staging URL, and evaluation date.
- Redact secrets and sensitive data before storing artifacts in this folder.
- Prefer exported artifacts (HTML, PDF, JSON, PNG) over screenshots when exports are available.

## How to fill

1. Record evaluation build details in `INDEX.md` before collecting evidence.
2. Place each report or output in the correct numbered folder and subfolder.
3. Update `INDEX.md` artifact links and checklist status for the current run.
