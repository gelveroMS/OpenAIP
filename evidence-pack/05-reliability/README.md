# 05 Reliability
## Purpose
Collect evidence for uptime, failure detection, and recovery readiness.
Use this folder for operational reliability outputs.
## Expected artifacts
- Availability exports in uptime/
- Error monitoring summaries in sentry/
- Recovery drill evidence in ecovery-drill/
## Naming convention
Use YYYY-MM-DD_<artifact>.<ext>, for example 2026-03-10_recovery-drill-report.md.
## Redaction reminder
Remove internal hostnames, credentials, and sensitive stack details before publishing.
