# 06 Security
## Purpose
Collect evidence for vulnerability checks, scan results, and access-control validation.
Use this folder for security testing outputs and audit traces.
## Expected artifacts
- Dependency audit exports in dependency-audit/
- DAST outputs in zap/
- Row-level security test evidence in ls-tests/
- Security-relevant audit extracts in udit-trail/
## Naming convention
Use YYYY-MM-DD_<artifact>.<ext>, for example 2026-03-10_zap-report.html.
## Redaction reminder
Do not commit secrets, tokens, private keys, or exploitable endpoint details.
